Pipeline

How It Works

A clean 4-stage pipeline that turns raw PDF documents into a queryable AI knowledge base with semantic search and context-aware answers.

01 / INGEST
📄
PDF Processing

PyPDF extracts raw text from documents. Supports multi-page academic papers, reports, and technical docs.

→
02 / CHUNK
✂️
Semantic Chunking

Documents split into contextually meaningful segments using semantic similarity — not just fixed character windows.

→
03 / EMBED
🧠
Vector Embeddings

OpenAI embeddings convert each chunk to high-dimensional vectors capturing deep semantic meaning.

→
04 / ANSWER
💬
RAG + GPT-4

Retrieval-Augmented Generation finds relevant chunks, feeds them as context to GPT-4 Mini for grounded answers.

Capabilities

Built for Accuracy,
Not Just Speed

Every feature is designed to minimize hallucination and maximize retrieval precision across complex technical documents.

📄
PDF-First Ingestion

Native PDF reading via PyPDF with proper handling of academic papers, multi-column layouts, and technical content.

🔬
Semantic Chunking

Unlike naive character-based splitting, semantic chunking preserves logical thought units — better context, better retrieval.

⚡
LanceDB Vector Store

Embedded vector database — no server setup needed. Fast ANN search across thousands of document chunks locally.

🧠
OpenAI Embeddings

State-of-the-art text embeddings capture semantic meaning so similar concepts retrieve even without keyword overlap.

🤖
Agno Agent Framework

Built on Agno — a lightweight, composable framework for building production AI agents with tool use and memory.

🛡️
Anti-Hallucination

Agent is explicitly instructed to reference only the knowledge base and say "I don't know" when information is unavailable.

What is the core idea behind the Transformer architecture?
🧠
The Transformer is built entirely on attention mechanisms, dispensing with recurrence and convolutions entirely. The key insight is that sequence transduction can be achieved through self-attention — allowing each position to attend to all positions in the encoder simultaneously, enabling much more parallelizable training.
📄 Source: attention_is_all_you_need.pdf · Section 3 (Model Architecture)
👤
What hardware was used for training?
🧠
Training was conducted on 8 NVIDIA P100 GPUs. Base models took approximately 12 hours (100,000 steps), while the big model variants required 3.5 days with 300,000 training steps.
📄 Source: attention_is_all_you_need.pdf · Section 5.2 (Hardware and Schedule)

# knowledge_base.py — Sets up the vector store and document pipeline from agno.knowledge.pdf import PDFKnowledgeBase from agno.vectordb.lancedb import LanceDb from agno.embedder.openai import OpenAIEmbedder from agno.document.chunking.semantic import SemanticChunking # 1. Configure the embedder (OpenAI text-embedding-3-small) embedder = OpenAIEmbedder() # 2. Semantic chunking — context-aware document splitting chunking_strategy = SemanticChunking( embedder=embedder, chunk_size=1000 ) # 3. LanceDB — serverless local vector database vector_db = LanceDb( uri="vector_db/lancedb", embedder=embedder, table_name="knowledge_table" ) # 4. Assemble the knowledge base with our PDF document knowledge_base = PDFKnowledgeBase( path="attention_is_all_you_need.pdf", vector_db=vector_db, chunking_strategy=chunking_strategy, )
# agent.py — Creates the AI agent with knowledge base access from agno.agent import Agent from agno.models.openai import OpenAIChat from knowledge_base import knowledge_base # Configure AI agent with GPT-4 Mini + knowledge base search agent = Agent( model=OpenAIChat(id="gpt-4o-mini"), knowledge=knowledge_base, search_knowledge=True, # Enables RAG retrieval instructions=[ "you are a helpful assistant", "whenever asked about the paper, use the knowledge base", "try not to hallucinate while responding", "if you don't know the answer, say I don't know", ], markdown=True, )
# main.py — Entry point: load KB and run queries from knowledge_base import knowledge_base from agent import agent # Load and index documents into vector database knowledge_base.load(recreate=False) # Set True to re-index # Run sample queries — general + knowledge-specific agent.print_response("Hi, how are you?") agent.print_response( "What hardware was used for training the transformer?" ) agent.print_response( "Explain the multi-head attention mechanism" )
Get Started

Quick Setup

Running locally in under 3 minutes with uv — the modern Python package manager. Requires Python 3.13+ and an OpenAI API key.

1
Clone & Install
$git clone https://github.com/Ashishverma66/
Agno-Knowledge_base-Chatbot
$cd Agno-Knowledge_base-Chatbot
$uv venv && source .venv/bin/activate
$uv sync
2
Configure API Key
$cp .env.example .env
$nano .env
# Add your OpenAI key:
OPENAI_API_KEY=sk-...
3
Run the Agent
# First run indexes the PDF (takes ~30s)
$python main.py
# To add your own PDFs, edit knowledge_base.py
# and set recreate=True on first load
4
Add Your Documents
# In knowledge_base.py, change the path:
path="your_doc.pdf"
# In main.py, set recreate=True once:
knowledge_base.load(recreate=True)
